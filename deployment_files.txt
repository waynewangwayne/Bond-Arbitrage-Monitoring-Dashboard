# README.md
# Bond Arbitrage Monitoring Dashboard

A comprehensive real-time dashboard for monitoring fixed income arbitrage opportunities, optimized for mobile access.

## Features

- **Real-time monitoring** of Primary-Secondary market spreads
- **When-Issued (WI) arbitrage** opportunity detection
- **Cash-Futures basis** trading signals
- **Z-score analysis** with statistical thresholds
- **Dark mode interface** optimized for mobile viewing
- **Automatic updates** every 30 seconds
- **Alert system** for high-probability trades

## Quick Start

### Local Development
```bash
git clone https://github.com/yourusername/bond-arbitrage-dashboard.git
cd bond-arbitrage-dashboard
pip install -r requirements.txt
streamlit run dashboard.py
```

### Streamlit Cloud Deployment

1. Fork this repository to your GitHub account
2. Go to [share.streamlit.io](https://share.streamlit.io)
3. Click "New app"
4. Select your repository: `yourusername/bond-arbitrage-dashboard`
5. Main file path: `dashboard.py`
6. Click "Deploy!"

Your dashboard will be available at: `https://yourusername-bond-arbitrage-dashboard-dashboard-main.streamlit.app`

## Configuration

### API Keys Setup

1. Get a FRED API key from [https://fred.stlouisfed.org/docs/api/api_key.html](https://fred.stlouisfed.org/docs/api/api_key.html)
2. In Streamlit Cloud, go to your app settings
3. Add secrets in the format:
```toml
FRED_API_KEY = "your_api_key_here"
```

## Troubleshooting

### Common Issues

1. **Slow loading on mobile**
   - Check internet connection
   - Clear browser cache
   - Use WiFi instead of cellular

2. **Data not updating**
   - Check API key configuration
   - Verify data source availability
   - Check Streamlit Cloud logs

3. **Charts not displaying**
   - Update browser
   - Enable JavaScript
   - Check mobile browser compatibility
